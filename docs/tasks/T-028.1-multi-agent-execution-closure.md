# T-028.1 - Multi-Agent Execution Closure

## Goal

Turn the existing M6 planner prototype into a deterministic Mock execution
slice: six registered agents must be scheduled by stage, workflow transitions
must be recorded, and the CLI must persist an execution artifact instead of a
plan-only manifest.

## Scope

- `runner_multi`, existing Coordinator state engine, Scheduler, agent registry,
  Mock execution, and focused tests.
- Stable run IDs and explicit failed execution results.

## Out of scope

- Live provider calls, dynamic topology, persistence, new tools, M6 closeout,
  or claims of full M6 acceptance.

## Acceptance

1. Mock CLI invokes all six registered agents through `MultiAgentScheduler`.
2. Artifact records stage results and workflow history.
3. Same input creates stable run ID across Python processes.
4. Revision exhaustion uses the configured inclusive limit.
5. Existing and new focused tests pass.

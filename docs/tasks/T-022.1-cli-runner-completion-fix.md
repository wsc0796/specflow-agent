# T-022.1 - CLI Runner Completion Fix

## Goal

Repair the T-022 delivery gap identified during the M5 review.  `specflow run`
must execute the existing Analyze, Generate, and Review Worker chain and write
the complete artifact set on a successful mock run.

## In scope

- Wire the existing three Workers into `specflow.runner` with valid prior-output
  contexts.
- Make `--max-files` configure evidence selection.
- Enforce the configured total evidence-character bound deterministically.
- Return documented CLI codes for provider configuration and review/degraded
  outcomes.
- Replace permissive CLI assertions with success-path and contract regression
  tests.
- Synchronize T-022 documentation and completion evidence.

## Out of scope

- T-023 repository evaluation, new Workers, new LLM providers, workflow API,
  persistence, RAG, embedding, or automatic code modification.

## Acceptance criteria

1. A `--provider mock` run reaches `COMPLETED` through Analyze, Generate, and
   Review and writes all ten artifacts.
2. `--max-files` limits the evidence collector's selected files.
3. Evidence excerpts never exceed `max_total_evidence_chars` in aggregate.
4. Missing OpenAI-compatible configuration returns exit code 2 without leaking
   secrets.
5. A completed run that is degraded or requires human review returns exit code 4.
6. Focused and full tests, Ruff check, Ruff format check, and `git diff --check`
   pass before commit.

## System model

| Dimension | Decision |
| --- | --- |
| Object | One candidate package version for the current `main` release candidate. |
| Truth source | `[project].version` in `pyproject.toml`; installed package metadata exposes it at runtime. |
| Lifecycle | `v1.0.1` remains published; `1.1.0` is an untagged main candidate until a user authorizes a release. |
| Invariants | OpenAPI, `specflow --version`, README and CHANGELOG all identify `1.1.0`; no source-only duplicate version constant is introduced. |
| Contracts | `specflow --version` exits 0 and prints the installed package version; `create_app().version` matches it. |
| Impact | Package entry point, app metadata, CI smoke command and current documents only. |

## Acceptance

- [ ] `pyproject.toml` declares `1.1.0`, while docs explicitly call it an
  unreleased candidate and preserve v1.0.1 as the latest published tag.
- [ ] `specflow.__version__`, FastAPI/OpenAPI version and CLI `--version` derive
  from the installed package metadata rather than hard-coded copies.
- [ ] Tests assert the exact candidate version, source-to-runtime equality,
  current README/CHANGELOG wording, and CLI output/exit code.
- [ ] CI performs the installed CLI version smoke command.
- [ ] Full quality gates pass; no tag/release is created.

## Testing subflow

future tag points at the expected commit; tagging/release publication remains a
separate user-authorized operation.
